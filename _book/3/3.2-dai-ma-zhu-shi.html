3.2 代码注释

3.2 代码注释

  1. 所有注释都应使用相同的语言编写，语法正确，并包含适当的标点。
  2. 使用 // 或 /// 但不要使用 /* ... */
  3. 不要 “flowerbox” 注释块。
    Example ： 
    // *************************************** 
// Comment block 
// ***************************************
  4. 使用内联注释来解释假设，已知问题和算法见解。
  5. 不要使用内联注释来解释明显的代码。好的代码自己就带注释。
  6. “fix this code” 只用来给坏代码注释 - 否则，删除或重写代码！
  7. 使用 Task-List 关键字标记注释以便筛选。
    Example ： 
    // TODO: Place Database Code Here 
// UNDONE: Removed P\Invoke Call due to errors 
// HACK: Temporary fix until able to refactor
  8. 始终将 C＃注释（///）应用于 public，protected，和 internal 声明。
  9. 仅使用 C＃ 注释块来解释 API。
  10. 始终包括 <summary> 注释。 包括 <param>，<return> 和 <exception> 注释适用的部分。
  11. 尽可能包括 <see cref =“”/> 和 <seeAlso cref =“”/>。
  12. 始终将 CDATA 标记添加到包含代码和其他嵌入式标记的注释中以避免编码问题。
    Example： 
    /// <example> 
/// Add the following key to the “appSettings” section of your config: 
/// <code><![CDATA[ <![CDATA[ 
///     <configuration> 
///         <appSettings> 
///             <add key=”mySetting” value=”myValue”/> 
///         </appSettings> 
///     </configuration> 
/// ]]>]] > </code> 
/// </example>
