4.3 流程控制

4.3 流程控制

  1. 避免在条件表达式中调用方法.
  2. 避免创建递归方法。 请改用循环或嵌套循环。
  3. 避免使用 foreach 迭代不可变的值类型集合。例如：字符串数组。
  4. 不要在 foreach 语句中修改枚举项。
  5. 仅对微不足道的条件使用三元条件运算符。 避免复杂或复合三元操作。
    Example：int result = isValid ? 9 : 4;

  6. 避免根据 true 或 false 判断布尔条件。
    Example：

    // Bad!
if (isValid == true) 
{…}

// Good!
if (isValid) 
{…}
  7. 避免在条件表达式中赋值和判断。
    Example：if((i=2)==2) {...}

  8. 避免使用复合条件表达式 - 使用布尔变量将部件拆分为多个可维护的表达式。 Example：

    // Bad!
if (((value > _highScore) && (value != _highScore)) && (value < _maxScore)) 
{…}

// Good!
isHighScore = (value >= _highScore); isTiedHigh = (value == _highScore); isValid = (value < _maxValue);

if ((isHighScore && ! isTiedHigh) && isValid) 
{…}

  9. 避免条件语中进行布尔值测试。
    Example：

    // Bad!
if (isValid == true) 
{…}

// Good!
if (isValid) 
{…}
  10. 仅使用 switch/case 语句进行简单的并行条件逻辑判断。
  11. 对于短条件序列和复杂条件， 首选嵌套 if/else 而非 switch/case。
  12. 倾向于使用 switch/case 的多样性来封装和委托复杂的操作。
